# ps 命令阅读指南

> ★ **ps 是 Linux 进程观测的基础工具** — 一次性快照所有进程的状态、资源使用和调度信息。
> 在性能分析方法论中，ps 是"进程状态检查"的起点，常与 top/pidstat 配合使用。

本目录包含 ps 命令的完整学习材料，从进程状态到命令参考再到实战输出。
按 01 → 03 的顺序阅读。

**预计总阅读时间：** 2-3 小时（快速入门 20 分钟，深入理解 1.5 小时）

## ★ 进程状态速查

| 状态 | 含义 | 常见场景 | 诊断方向 |
|------|------|---------|---------|
| **R** (Running) | 运行中或就绪 | 正常 | CPU 充足时正常 |
| **S** (Sleeping) | 可中断等待 | 等待事件/信号 | 正常状态 |
| **D** (Disk sleep) | 不可中断等待 | ★ I/O 阻塞、NFS 卡死 | → iostat/blktrace |
| **Z** (Zombie) | 僵尸进程 | 父进程未 wait() | 检查父进程 |
| **T** (Stopped) | 暂停 | SIGSTOP/SIGTSTP | 检查调试器 |

## ★ 与其他工具的交叉引用

### ps → pidstat — 从快照到持续监控

```
1. ps aux → 一次性快照，发现异常进程（D 状态、高 CPU、Zombie）
   │
   ├── D 状态进程多 → I/O 阻塞
   ├── %CPU 高 → 计算密集
   └── Zombie 多 → 父进程问题
   │
   ▼
2. pidstat -u -d -w -p <PID> 1 → 持续监控该进程的资源使用
   → 量化 CPU/I/O/内存/上下文切换的每秒速率
```

### ps → top/htop — 从快照到实时

```
1. ps aux → 初始快照
2. top → 动态观察进程变化
3. 对比两次 ps 快照 → 计算资源变化速率
```

### ps → perf — 从状态到原因

```
1. ps aux → 发现 %CPU 最高的进程
2. perf record -p <PID> -g → 采集函数级热点
3. perf report → 火焰图定位 CPU 消耗在哪个函数
```

### ps → iostat/blktrace — D 状态排查

```
1. ps aux → 发现大量 D 状态进程
2. iostat -x 1 → 检查设备 %util 和 await
3. blktrace → 追踪 I/O 延迟细节
4. cat /proc/<PID>/wchan → 进程卡在内核的哪个函数
```
