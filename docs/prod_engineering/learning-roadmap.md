# 生产级软件工程方法论

> 从"功能能跑"到"系统可靠"——构建生产级存储/系统软件必须考虑的维度

---

## 为什么需要这一节？

前面 17 个工具教你**如何观测和诊断**已有的系统。但当你**自己设计实现**一个存储软件、文件系统、数据库或系统服务时，仅保证功能正确远远不够。生产环境会遭遇：

```
功能实现完成 ←──────────── 你在这里（只是起点）
    │
    ├── 崩溃了怎么办？数据会不会丢？
    ├── 升级时能不能不中断服务？
    ├── 异常如何分类处理？重试还是放弃？
    ├── 出了问题怎么排查？日志够不够？
    ├── 性能会不会在压力下雪崩？
    ├── 被恶意输入攻击怎么办？
    ├── 资源耗尽怎么办？OOM？fd 耗尽？
    └── 多节点如何保持一致？
    
生产级可靠 ←──────────── 目标在这里
```

本节覆盖 8 个维度，每个维度包含**背后思想、常用实现、坑与权衡**：

## 学习建议

### 如果你是存储/文件系统开发者
1. 先读 `01_crash_recovery`（fsync 语义是存储软件的根基）
2. 再读 `02_hot_upgrade`（用户态文件系统如 FUSE 的升级）
3. 然后 `05_concurrency_perf`（I/O 路径的锁）

### 如果你是分布式系统开发者
1. 先读 `08_consistency`（CAP、共识、fencing）
2. 再读 `04_fault_tolerance`（重试、超时、幂等）
3. 然后 `03_observability`（分布式追踪）

### 如果你是系统服务开发者
1. 先读 `04_fault_tolerance`（信号处理、优雅退出）
2. 再读 `06_resource_management`（fd/内存泄漏）
3. 然后 `07_security`（权限分离、沙箱）
